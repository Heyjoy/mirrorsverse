# Mirrorsverse — Impeccable Design Context

> Persistent design context for mirrorsverse.com. All design/frontend work in this repo should respect this file.

## Design Context

### Users
站点同时服务四类访客，文案与信息架构需在同一页面内分层满足：

1. **潜在 B2B 客户 / 分包方** — 评估 AI Agent 开发能力、案例、报价路径，关注「能交付什么」「做过什么」
2. **HR / 招聘方** — 验证公司正规度、技术实力、创始人背景，关注信任信号
3. **技术同行 / 开发者** — 看技术深度、架构思路、开源/技术内容，关注「技术含量」
4. **中文聚会 / 线下场景** — 通过 `/card` 名片页 3 秒内建立联系，关注二维码与核心一句话

**核心 job-to-be-done**: 在 30 秒内让访客判断「这是一家有真实技术能力的独立 AI 服务公司」并留下联系方式。

### Brand Personality
**前卫 · 实验 · 锐利**（Avant-garde · Experimental · Sharp）

- **Voice**: 第一人称克制、技术词汇精准、避免营销腔；中文不翻译腔，英文不堆 buzzword
- **Tone**: 自信但不傲慢，技术细节拿得出，留白处不解释
- **Emotional goal**: 让访客产生「这家有点不一样 / 想看看他们做了什么」的好奇感，而非「又一个 AI 公司」的疲劳感

### Aesthetic Direction

**Visual tone**: 深色为主、冷蓝紫 accent（已有 oklch 0.65 0.15 270 基线）、几何/网格底纹、大字号 display 排版、不对称布局、有节制的动效

**双语**: 中英并列时不让中文当英文附属——CJK 与 EN 字体配对、字号节奏独立调

**参考方向**（情绪锚点，不是抄）:
- Linear / Vercel — 技术克制
- Rauno.me / Basement.studio — 实验排版与动效
- Anthropic / OpenAI 的工程博客 — 信息密度与中性灰调

**Anti-references（明确不要）**:
- ❌ 通用 AI SaaS 模板：紫渐变 + emoji + "Powered by AI" hero + 三栏 feature card
- ❌ 传统中式企业站：蓝色顶栏 + 公司大事记 + 资质轮播 + 滚动文字横幅
- ❌ 极简到无信息：只有一行字 + 一个按钮的「装酷」留白派

**主题**: Dark first（亮色可后续补，不优先）

### Design Principles

1. **信息密度 > 留白炫技** — 访客有 30 秒，每屏要承载真实内容（数字、案例、技术词），不做空气页
2. **不对称胜于对称** — 拒绝 3 列等宽 card grid，用 bento / asymmetric / 错位排版建立辨识度
3. **动效服务可读性** — 动画用于引导视线和层次（fade-in-up、stagger、scroll-trigger），不做装饰性 WebGL 满屏；尊重 `prefers-reduced-motion`
4. **中文是一等公民** — `/zh` 与 `/en` 同等优先，CJK 字体配对、行高、标点都要单独调，不直接套英文样式
5. **WCAG AA 是底线** — 对比度 ≥ 4.5:1（深色主题尤其要验）、键盘可达、focus ring 可见、动效可降级
